# What is blockcell?

> A self-evolving AI agent framework written in Rust

---

## Starting with a Real Pain Point

You've probably been through this:

You ask ChatGPT a question. It gives you a Python script. You manually copy it to your editor, run it, get an error. You paste the error back, it fixes the code, you copy again, run again...

Or you want AI to send you a stock market summary every morning at 8 AM. But your AI tool can't run on a schedule, can't access real-time data, and can't push notifications to your phone.

**AI "knows" how to do these things, but it can only talk, not act.**

blockcell is built to solve exactly this problem.

---

## What blockcell Is

blockcell is an open-source **AI agent framework** written in **Rust**.

It makes AI not just conversational, but capable of **actually executing tasks**:

```
You say: "Analyze sales.xlsx on my desktop, create a line chart, and send it to my Telegram"

blockcell:
  1. Reads ~/Desktop/sales.xlsx
  2. Analyzes data with Python, calculates trends
  3. Generates line chart with matplotlib → chart.png
  4. Sends the image to you via Telegram Bot
  ✅ Done — zero manual steps required
```

## Core Design: Two-Layer Architecture

blockcell uses a stable "host layer" and a flexible "skill layer":

```
┌──────────────────────────────────────────────────────┐
│           Rust Host Layer (Trusted Computing Base)    │
│                                                      │
│  Message Bus  │  Tool Registry  │  Task Scheduler    │
│  Security     │  Provider Factory│  Subagent Engine  │
│  Memory Store │  Self-Evolution  │  Audit Log        │
└──────────────────────────────────────────────────────┘
                        ↕  Standard Interface
┌──────────────────────────────────────────────────────┐
│           Rhai Skill Layer (Flexible, Evolvable)      │
│                                                      │
│  stock_monitor  │  bond_monitor  │  daily_report     │
│  crypto_onchain │  browser_task  │  your custom skill│
└──────────────────────────────────────────────────────┘
```

**The Rust host layer** is the stable core: handles security, performance, scheduling, and tool execution. It doesn't change arbitrarily — it's the trust anchor of the entire system.

**The Rhai skill layer** is the flexible extension: written in a scripting language, skills can be added, modified, or even auto-generated by AI at any time.

Think of it like a smartphone: stable OS kernel (Rust host) + flexible apps (Rhai skills).

## The Most Unique Feature: Self-Evolution

When a skill repeatedly fails, blockcell **automatically repairs itself**:

```
1. Track error patterns (threshold triggers evolution)
2. LLM analyzes the issue and generates a new version
3. Automatic code safety audit
4. Syntax / compilation check
5. Canary rollout: 10% traffic → 50% → 100%
6. Monitor error rate — auto-rollback if worse
```

This means blockcell becomes more reliable the more you use it, like an **employee who learns from mistakes**.

## Why Rust?

Many ask: shouldn't an AI framework be Python?

blockcell chose Rust for concrete reasons:

1. **Memory safety**: The host layer won't crash from unexpected memory issues during AI execution
2. **Concurrency**: Run many subagents simultaneously, no Python GIL bottleneck
3. **Trustworthy base**: The host is the "Trusted Computing Base" — it must be stable and predictable
4. **Single binary**: Compiles to one file, runs on macOS / Linux / Windows with no dependencies
