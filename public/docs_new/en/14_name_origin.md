# Name Origin

> Block + Cell: A stable foundation and evolvable capability units

---

## Why "blockcell"?

The name is a compound of two words: **Block** and **Cell**.

Each represents one of the framework's two most fundamental design principles.

---

## Block: The Stable Foundation

**Block** evokes images of building blocks, foundation stones, and fundamental units of structure.

In blockcell's architecture, **Block represents the Rust host layer** — the stable, trustworthy, unchanging core.

Like the baseplate of a LEGO set:
- Always there, always stable
- Defines the rules and interfaces
- Anything can be built on top of it
- It doesn't rearrange itself

The choice of Rust reinforces this. Memory safety, no crashes, a vast class of bugs eliminated at compile time. This "foundation" is the system's **Trusted Computing Base (TCB)**.

> "You don't need to trust every AI decision. But you do need to trust the infrastructure that executes those decisions."

---

## Cell: The Evolvable Unit

**Cell** evokes a biological cell — the basic unit of life, capable of division, mutation, and adaptation.

In blockcell, **Cell represents the Rhai skill layer** — the flexible, replaceable, self-evolving extension layer.

Like biological cells:
- Each skill is an independent "cell"
- Can be replicated (install new skills)
- Can mutate (self-evolve, generate new code versions)
- Can die (disable or delete a skill)
- The system overall maintains vitality (continuous improvement)

---

## The Combination

**Block + Cell = Evolvable capability units on a stable foundation**

This combination describes an **architectural philosophy**:

```
The core (Block) stays stable — unaffected by AI randomness
  +
The capabilities (Cell) stay flexible — always extendable and self-repairing
  =
An AI agent framework that is both reliable and self-evolving
```

---

## The Blockchain Connection

You might think of "blockchain." blockcell is not blockchain technology, but the association isn't entirely accidental.

Blockchain emphasizes **immutability** and **decentralized trust**. blockcell emphasizes a **trusted execution environment** and **transparent tool invocation** (every AI tool call exposes its arguments and results to you).

Both are using technology to answer the same fundamental question: **How do you build a foundation of trustworthy computation?**
