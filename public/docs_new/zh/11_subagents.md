# 子智能体与任务并发

> 让多个 AI 同时工作，就像有一支 AI 团队

---

## 什么是子智能体？

普通的 AI 对话是**串行**的：AI 做完一件事，才开始下一件。

子智能体（Subagent）系统允许你**并行**启动多个独立的 AI 任务，它们在后台同时运行，互不干扰。

**类比**：这就像你作为项目经理，把不同任务分配给团队成员，然后继续做别的事。等他们完成了，来汇报结果。

---

## 何时使用子智能体？

适合这些场景：

- **耗时任务**：数据抓取、大量文件分析、视频处理
- **并行任务**：同时监控多支股票、同时处理多个数据源
- **周期性任务**：每小时执行一次的后台工作
- **独立任务**：互相不依赖、可以同时进行的多件事

不适合：
- 需要顺序执行、结果有依赖关系的任务（用技能流程代替）
- 简单的一次性查询（直接对话即可）

---

## 启动子智能体

在对话中，用 `spawn` 工具启动后台任务：

```
你: 帮我同时做这三件事：
  1. 分析工作目录里的所有 .py 文件，找出代码复杂度最高的前 5 个函数
  2. 搜索最近一周关于 Rust 和 AI 的技术文章，整理成摘要
  3. 查询我持仓的 5 支股票的最新行情
  都完成后分别告诉我结果
```

AI 会为每个任务 spawn 一个子智能体，同时开始执行。

### 简单的单任务

```
你: 帮我在后台分析一下工作目录里所有的日志文件，找出出现频率最高的错误，完成后告诉我
```

## 任务管理

### 查看任务状态

```
You: /tasks
```

或：

```
你: 现在有哪些后台任务在运行？
```

输出示例：

```
后台任务：

[运行中] task_abc123
  📋 分析 Python 文件代码复杂度
  ⏱ 已运行 2 分 15 秒
  📍 正在处理 utils/helpers.py...

[已完成] task_def456
  📋 搜索 Rust + AI 技术文章
  ✅ 完成于 3 分钟前

[排队中] task_ghi789
  📋 查询持仓股票行情
  ⏳ 等待执行
```

### 通过 CLI 管理

```bash
# 查看所有任务
blockcell tasks

# 查看特定任务详情
blockcell tasks show task_abc123

# 取消任务（如果支持）
blockcell tasks cancel task_abc123
```

## 任务完成通知

子智能体完成时，会通过你配置的渠道通知你：

- 如果在终端里 → 在当前对话中显示结果
- 如果配置了 Telegram → 发送 Telegram 消息
- 如果在 Gateway 模式 → 通过 WebSocket 推送事件

---

## 子智能体的能力限制

子智能体拥有完整的工具能力，但有一些限制（为了安全）：

| 能力 | 子智能体 |
|------|---------|
| 读写文件 | ✅ |
| 执行命令 | ✅ |
| 网络请求 | ✅ |
| 访问记忆 | ✅ |
| 发消息通知 | ✅ |
| 再次 spawn 子智能体 | ❌（防止无限嵌套） |
| 修改定时任务 | ❌ |

这个设计防止了子智能体失控，同时保留了完整的工作能力。

## 实战案例

### 案例 1：批量数据分析

```
你: 工作目录里有 20 个 CSV 数据文件（sales_jan.csv 到 sales_dec.csv），
  帮我：
  1. 为每个文件生成一个简单的统计摘要（行数、主要列、数据质量）
  2. 找出哪些月份的销售额最高
  3. 画一张全年销售趋势图
  
  不用等每一步，可以并行处理
```

### 案例 2：投资研究

```
你: 帮我同时研究这 3 支股票，每支都要：查最新价格、看技术指标、搜一下最新的分析师评级

茅台（600519）、宁德时代（300750）、比亚迪（002594）
```

### 案例 3：内容生成流水线

```
你: 帮我：
  1. 搜索今天 AI 领域最重要的 5 条新闻
  2. 同时搜索今天区块链领域的重要新闻
  3. 等两部分都搜完后，整合成一篇早报，发到我的 Telegram
```

注意：步骤 1 和 2 可以并行，步骤 3 需要等待 1 和 2 完成。这需要在对话中说清楚。

## TaskManager 内部机制

每个任务都由 `TaskManager` 管理，记录：

| 字段 | 说明 |
|------|------|
| `id` | 任务 ID（唯一标识） |
| `label` | 任务描述 |
| `status` | Queued / Running / Completed / Failed |
| `progress` | 当前进度描述 |
| `result` | 完成后的结果摘要 |
| `origin_channel` | 来源渠道（用于回复到正确的地方） |
| `created_at` | 创建时间 |
| `completed_at` | 完成时间 |

任务在完成后会在 TaskManager 中保留一段时间（约 5 分钟），然后自动清理。

## 子智能体使用独立的 AI 实例

每个子智能体都是一个完整的、独立的 `AgentRuntime` 实例：
- 独立的对话历史
- 独立的工具调用循环
- 使用相同的 LLM 配置（可以指定不同模型）
- 并发执行，互不阻塞

---

## 与 cron 定时任务的区别

| 特性 | spawn 子智能体 | cron 定时任务 |
|------|--------------|-------------|
| 触发方式 | 手动 | 定时自动 |
| 执行次数 | 一次 | 周期性 |
| 适合场景 | 即时并行任务 | 定期重复任务 |
| 示例 | 批量分析文件 | 每天 8 点发日报 |

两者可以配合使用：cron 任务里可以 spawn 子智能体处理复杂的并行子任务。
